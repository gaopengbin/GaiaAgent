### 2. geocode "未知指令" 根因与修复
- **现象**: 前端调 geocode 返回 `{"success":false,"error":"未知指令: geocode"}`
- **根因**: `/api/command` 是 fire-and-forget，所有命令盲推到浏览器 bridge，bridge 无 geocode handler
- **修复 1**: cesium-mcp-runtime@1.139.19 新增 `SERVER_SIDE_TOOLS` 集合，HTTP 端点拦截服务端工具
- **修复 2**: Rust `call_tool` 拦截 geocode，直接调 `geocode_nominatim()` 绕过 bridge

### 3. Nominatim 网络不通 → native-tls + 代理
- rustls-tls 无法连接 Nominatim（法国 88.191.249.183），PowerShell 可以
- 尝试 Tianditu geocode API → 连接正常但 403（browser-type token 限制）
- reqwest 从 `rustls-tls` 切换为 `native-tls`（Windows SChannel）→ 仍不通
- 最终方案：**ModelSettings 新增 `proxyUrl` 字段**，geocode 走用户配置的本地代理
- `geocode_nominatim()` 在 proxyUrl 非空时构建 `reqwest::Proxy::all()` 代理客户端

## 关键决策
- **放弃 Tianditu geocode**: token 类型限制导致非浏览器 403，不可行
- **保留 native-tls**: 虽然单独不能解决 Nominatim 问题，但比 rustls 更贴近系统行为
- **代理为可选配置**: proxyUrl 为空时直连，非空时走代理，不强依赖
